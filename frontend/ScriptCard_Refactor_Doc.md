# 剧本卡富文本编辑器集成说明

## 1. 概述
为提升用户的剧本编辑体验，将原有基于弹窗的表单编辑模式重构为了卡片内嵌直接文本编辑。该实现采用了开源的 `Tiptap` 作为底层富文本编辑器，并使用其原生支持的 JSON 格式来存储内容以确保数据的结构完整性和未来可扩展性。

## 2. 变更说明
- **依赖库增加**: 安装了 `@tiptap/react`, `@tiptap/starter-kit`, `@tiptap/extension-placeholder` 用于实现卡片内部的富文本编辑功能。
- **状态存储调整**: `useCanvasStore.ts` 中的 `ScriptNodeData` 接口增加了 `content?: any` 字段，以存储 Tiptap 导出的 JSON 对象。兼容旧数据，对之前的 `description` 也进行了向前兼容处理。
- **组件重构**: 
  - 移除了原有的 `ScriptEditModal` 弹窗组件。
  - 在 `ScriptNode.tsx` 内部直接集成编辑状态 (`isEditing`)。通过状态切换提供平滑的交互：编辑、保存和取消。
  - 增加内联标题输入框及相关标签与角色的编辑入口。
- **新增组件 `ScriptEditor`**: 基于 Tiptap 封装了通用富文本编辑器组件，内置了加粗、斜体、删除线及列表等基础排版能力。

## 3. 使用说明

1. **进入编辑模式**
   - 用户点击卡片右下角的“编辑”按钮即可将卡片切换为编辑状态。此时卡片会适当放大并在页面中置顶。

2. **内容格式化**
   - 编辑器顶部会展示一个迷你工具栏，用户可以通过点击对应按钮快速应用或取消样式（加粗、斜体、删除线、有序/无序列表）。
   - 支持标准的快捷键（如 `Ctrl/Cmd + B` 加粗）。

3. **保存与取消**
   - 点击右下角的“取消”按钮会撤销所有在当前编辑会话中的修改。
   - 点击右下角的“保存”按钮会将当前数据实时同步至全局 `Zustand` Store 中并保存。

## 4. 数据双向绑定与兼容
- 初始化时，如果不存在 `content` 字段但存在旧版的 `description` 字段，会自动将其转换为 Tiptap 可识别的 JSON 结构并渲染。
- 用户输入时，Tiptap 的 `onUpdate` 钩子会调用 `editor.getJSON()` 将最新的文档结构传递回父组件 `ScriptNode` 中，并在保存时同步到全局状态。

## 5. 测试
- 已在 `src/components/canvas/__tests__` 下补充了对应的单元测试：
  - `ScriptNode.test.tsx`: 覆盖了节点的渲染、编辑模式的切换、数据的保存及取消、节点删除确认逻辑。
  - `ScriptEditor.test.tsx`: 覆盖了编辑器的渲染逻辑和只读/编辑状态下工具栏的显示隐藏。
- 可通过运行 `npx jest src/components/canvas/__tests__` 验证这些指标。
