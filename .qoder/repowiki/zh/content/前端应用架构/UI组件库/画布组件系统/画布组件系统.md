## 简介
本文件系统性梳理 Infinite Game 项目中的画布组件体系，围绕基于 React Flow 的可视化画布进行深入解析。内容涵盖画布节点（角色、剧本、故事板、视频）的设计与实现、连接线系统的定制、侧边栏工具面板与属性编辑器、以及拖拽、缩放、对齐等交互功能。特别关注最新的UI改进与重构，包括移除不必要的CSS类、简化组件结构以及VideoNode的资产元数据处理增强。同时提供使用示例与扩展开发指南，帮助开发者快速理解并二次开发。

## 项目结构
画布系统主要位于前端应用的剧院编辑页面，采用模块化组织：
- 页面容器：剧院编辑页负责初始化 React Flow、注册节点与边类型、挂载侧边栏与控制面板
- 节点组件：四种节点分别承载不同媒体与内容形态
- 连接线组件：自定义贝塞尔曲线边，支持悬停高亮与删除
- 样式系统：统一的SCSS模块管理edge handle样式，提升代码可维护性
- 存储与状态：Zustand 状态管理，持久化本地存储，支持后端同步
- 工具与钩子：拖拽、吸附、自动布局、图算法等辅助能力

## 架构总览
画布系统采用"页面容器 + React Flow + 组件层 + 状态层 + 工具层"的分层架构。页面容器负责装配与编排，React Flow 提供基础画布能力，组件层实现节点与边的具体渲染与交互，状态层统一管理节点、边、视口与后端同步，工具层提供拖拽、吸附、布局等增强能力。最新的样式系统统一通过SCSS模块化管理，确保所有节点的edge handle样式一致性和可维护性。

## UI改进与重构

### 组件简化重构
最新的UI改进专注于移除不必要的CSS类并简化组件结构，同时保持视觉一致性。这一重构主要体现在以下几个方面：

#### CharacterNode组件优化
- **移除冗余CSS类**：清理了不再使用的CSS类，减少了DOM复杂度
- **结构简化**：优化了节点包装器的类名使用，确保只保留必要的样式类
- **性能提升**：减少DOM节点数量，提升渲染性能

#### NodePreviewCard组件改进
- **样式精简**：移除了不必要的CSS类，保持组件简洁性
- **视觉一致性**：通过统一的样式系统确保预览卡片与主界面风格一致

## 样式系统统一

### SCSS模块化管理
画布组件系统实现了样式系统的全面统一，所有节点的edge handle样式从复杂的inline CSS迁移到统一的SCSS模块管理。这种变更带来了以下优势：

- **集中管理**：edge handle样式集中在 `_edge-handle.scss` 文件中，便于维护和修改
- **一致性**：所有节点共享相同的样式规范，确保视觉一致性
- **可扩展性**：新增节点类型时无需重复编写样式代码
- **主题支持**：通过SCSS变量和混合器支持深色/浅色主题切换

### 统一样式架构
新的样式系统采用模块化架构，主要包括：

#### 基础样式类
- `.edge-handle-wrapper`：拖拽手柄包装器的基础样式
- `.edge-handle-inner`：内部装饰元素的通用样式
- `.edge-handle-line`：连接线的基础样式
- `.edge-handle-dot`：圆点装饰的基础样式

#### 定位系统
- 使用百分比定位而非固定像素值，避免节点高度不确定时的计算错误
- 左右边缘分别使用不同的定位策略，确保在各种节点尺寸下的正确对齐

#### 交互状态
- 悬停状态：通过 `:hover` 伪类控制透明度变化
- 激活状态：连接手柄高亮显示，使用特定的颜色和边框
- 指针事件：精确控制鼠标事件的传递和捕获

## 性能考虑
- 虚拟渲染与懒加载：节点内容（如视频）在可见时才加载，减少初始渲染压力
- 状态更新批处理：批量更新节点位置与尺寸，避免频繁重渲染
- 布局算法优化：Dagre 布局仅在必要时触发，且通过过渡动画平滑位置变化
- 拖拽与吸附：吸附计算仅在拖拽过程中进行，停止拖拽后清空对齐线，降低 DOM 开销
- 缓存与去重：状态持久化合并时去重节点，避免重复渲染
- 样式系统优化：SCSS模块化减少CSS重复，提升渲染性能
- 组件简化：移除不必要的CSS类，减少DOM复杂度，提升渲染性能

## 故障排除指南
- 无法连接：检查是否自环或会形成环（环检测会阻止），确认连接手柄方向
- 上传失败：检查后端上传接口可达性、认证头、文件类型与大小限制
- 节点尺寸异常：确认图片/视频元数据获取成功，尺寸计算逻辑是否触发
- 拖拽无效：确认 dataTransfer 中携带了正确的类型、数据与尺寸信息
- 自动布局错乱：检查节点测量尺寸是否正确，Dagre 参数是否合适
- 样式不生效：检查SCSS文件是否正确导入，CSS类名是否匹配
- edge handle显示异常：确认节点包装器类名和Handle组件的正确嵌套
- 资产元数据同步失败：检查VideoNode的asset元数据处理逻辑和资源库同步方法
