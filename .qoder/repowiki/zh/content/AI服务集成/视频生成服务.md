## 简介
本文件系统性阐述多供应商视频生成服务的集成架构与实现细节，覆盖 xAI Grok Video、MiniMax 海络、Gemini Veo、火山方舟等视频生成服务的统一适配器设计。文档重点说明 VideoProviderAdapter 基类、VideoContext/VideoResult 数据结构、视频生成上下文管理与结果处理流程；并给出 API 调用示例、进度跟踪机制与质量控制策略，解释如何配置不同视频生成参数、处理生成队列与优化生成性能。**新增功能**：支持多模态参考输入，包括图片、视频、音频的组合生成，以及首帧/尾帧过渡功能。

## 项目结构
视频生成服务位于后端目录 backend，采用"适配器 + 工厂 + 路由"的分层架构：
- 适配器层：video_providers 目录下提供统一抽象 VideoProviderAdapter 及各供应商适配器（xAI、MiniMax、Gemini、火山方舟）
- 服务层：video_generation.py 提供统一入口与轮询逻辑
- 路由层：routers/videos.py 提供 REST API，对接数据库与计费
- 能力配置：model_capabilities.py 定义各模型能力矩阵
- 类型与配置：schemas.py 定义请求/响应模型与 LLMProvider 配置
- 技能工具：video_tools 目录提供用户界面和参数配置

## 多模态参考输入功能

### 功能概述
多模态参考输入功能允许用户同时使用图片、视频、音频等多种媒体类型进行视频生成，实现更丰富和精确的创作效果。该功能支持以下特性：

- **参考图片数组**：最多 9 张图片，用于提供视觉风格、角色特征、场景元素等参考
- **参考视频数组**：最多 3 个视频，用于提供运动轨迹、镜头语言、动作参考等
- **参考音频数组**：最多 3 个音频，用于提供音乐风格、音效特征、节奏参考等
- **首帧/尾帧过渡**：实现视频的自然开始和结束效果
- **多模态组合**：支持图片+视频+音频的混合参考，满足复杂的创作需求

### 技术实现

#### VideoContext 扩展
```python
@dataclass
class VideoContext:
    # ... 其他字段 ...
    
    # 参考图片 (Grok / Gemini Veo 3.1 支持, 最多 3 张)
    reference_images: List[Dict] = field(default_factory=list)
    
    # 参考视频 (Seedance 2.0 支持, 最多 3 个)
    reference_videos: List[Dict] = field(default_factory=list)
    
    # 参考音频 (Seedance 2.0 支持, 最多 3 个)
    reference_audios: List[Dict] = field(default_factory=list)
    
    # 首帧图片 (xAI/Gemini 支持)
    image_url: str | None = None
    
    # 尾帧图片 (xAI/Gemini 支持)
    last_frame_image: str | None = None
```

#### 适配器支持情况
- **xAI Grok Video**：完全支持多模态参考输入，支持 reference_images、reference_videos、reference_audios
- **Gemini Veo**：支持参考图片和视频扩展，支持最多 3 张参考图片
- **火山方舟 Seedance 2.0**：全面支持多模态参考输入，支持最多 9 张参考图片、3 个参考视频、3 个参考音频
- **MiniMax**：当前主要支持首帧图片，多模态参考输入支持有限

### 使用示例

#### 基础多模态参考

## 性能考虑
- 轮询间隔与超时：统一入口 MAX_POLL_FAILURES 控制轮询失败阈值；路由层对 pending 且带错误超过 5 分钟判定失败，避免长时间占用
- 资源映射：MiniMax 分辨率映射、Gemini 宽高比/分辨率映射、火山方舟分辨率/比例映射，减少供应商不支持参数导致的重试
- I/O 优化：MiniMax 完成后立即调用 get_video_url 获取下载链接；Gemini 完成后使用 API Key 直接下载
- 模型能力预检：通过 model_capabilities 预先校验模型能力，避免无效请求
- **多模态优化**：合理使用参考媒体数量，避免过多参考导致生成时间过长

**更新**：多模态参考输入可能增加生成时间和资源消耗，建议根据具体需求合理选择参考媒体数量

## 故障排查指南
- 提交失败：统一入口返回 status=failed 时，路由层抛出 502 并携带错误信息
- 轮询失败：MAX_POLL_FAILURES 控制失败次数；路由层对 pending 且带错误超过 5 分钟判定失败
- MiniMax 下载：完成状态需调用 get_video_url 获取下载链接；若为空，检查 file_id 与 API Key
- 内容审核：xAI 审核拒绝映射为 failed；检查 moderation_ok 字段
- 错误消息提取：火山方舟错误消息格式多样，统一通过 _extract_error_message 提取
- 模型能力不匹配：通过 model_capabilities 校验模型支持的模式与时长/分辨率等
- **多模态参考问题**：检查参考媒体格式、大小限制，确保 URL 可访问性

**更新**：新增多模态参考输入相关的故障排查指导

### 多模态参考输入最佳实践

#### 参考媒体选择策略
- **图片参考**：选择高质量、清晰度高的图片，避免模糊或过度压缩的图片
- **视频参考**：选择与目标视频风格相似的视频片段，长度控制在 3-10 秒之间
- **音频参考**：选择与视频内容相符的背景音乐或音效，避免版权问题

#### 数量控制建议
- **图片参考**：1-3 张即可满足大多数场景需求
- **视频参考**：1-2 个视频片段，避免过多参考导致生成效果混乱
- **音频参考**：1 个音频文件，确保音频与视频内容协调

#### 格式兼容性
- **图片格式**：JPG、PNG、WEBP 等主流格式
- **视频格式**：MP4、AVI、MOV 等常见格式
- **音频格式**：MP3、WAV、AAC 等常用格式
